Preguntas a responder

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

El Internet es una red que conecta a las computadoras, mientras que la World Wide Web son páginas que se visualizan mediante una conexión a Internet. Por lo tanto, es necesario acceder a Intenet para ver la información de la WWW.

2. ¿Cuáles son las partes de un URL?

Un URL cuenta con protocolo de transmisión, nombre del host, subdominio, dominio, Top Level Domain o TLD, ruta, archivo en el directorio y/o parémetro.

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Los metodos de petición de HTTP funcionan para que los clientes se comuniquen con el servidor:

  • GET: Se utiliza para solicitar información.
  • HEAD: Se utiliza para solicitar al servidor únicamente el encabezado de un archivo.
  • POST: Manda datos al servidor
  • PUT: Crea o reemplaza a un recurso.
  • PATCH: Modifica parcialmente a un recurso
  • DELETE: Borra un recurso.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se utiliza el método POST porque esta petición no muestra los datos en la barra de direcciones como el método GET.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET porque el cliente solicita que el servidor mande el archivo de la página para poder visualizarla.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No ha ocurrido un error. El código indica que la petición fue exitosa.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No es responsabilidad del desarrollador, a menos que el link esté roto. El error 404 quiere decir que la página no fue encontrada por el servidor.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, este error significa que hubo un error en el servidor de la página y debe de ser arreglado por el desarrollador.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo despreciado es uno que ya no es recomendado en nuevas versiones de HTML porque perjudican la usabilidad o accesibilidad. Por ejemplo acronym, applet, basefont y big.

10. ¿Cuáles son las diferencias principales entre HTML4 y HTML5?

HTML5 es más simple que HTML4 ya que su sintaxis se hizo más sencilla para facilitar su uso. Además, HTML5 soporta multimedia sin necesidad de un tercero y contiene más etiquetas que HTML4.

11. ¿Qué componentes de estructura y estilo tiene una tabla?

Las tablas son una colección de celdas que se organizan en filas y columnas. Su estructura se puede definir con las etiquetas table, tr, th y td.

12. ¿Cuáles son los principales controles de una forma HTML5?

Los controles de una forma son comandos que permiten al usuario introducir datos para mandarlos al servidor. Algunos son botones, cajas de texto, casillas, etc. Las etiquetas de los controles son input, button, select, option, optgroup y textarea.
